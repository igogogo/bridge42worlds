# Developer handoff — рабочие блоки (bridge42worlds)

Составлено 2026-07-21 по итогам Quality Check. Каждый блок = один связный PR, который можно
сделать целиком и консистентно. Внутри блока: **где смотреть** (файлы), **что сделать** (шаги),
**как проверить** (репро + ожидаемый результат + команда). Архитектурные задачи (гамбургер,
sticky-шапка, языковой глобус, theory→education, «тег»→«понятие», 3 раздельных поля поиска) сюда
НЕ входят — они на отдельное решение владельца.

## Общий рабочий цикл (для любого блока)

```bash
# 1. правишь templates/*.html, js/*.js, css/style.css, generate.py
# 2. офлайн-пересборка HTML (без API, ~10 мин, 20912 стр.):
python run.py html
# 3. проверка внутренних ссылок на 404:
python run.py links
# 4. проверка целостности:
python run.py check
# 5. локальный просмотр:
python -m http.server 8912   # → http://localhost:8912/lang/ru/index.html
```
`asset_ver()` сам версионирует css/js (sha256 → `?v=`), кэш чистить не нужно. НЕ трогай API-команды
(`regen`, `daily`, `translate-one`, `bulk-*`, `images --gen-images`) — они платные; блоки ниже чисто
офлайновые (шаблоны/CSS/JS/рендер).

## Разделение работ и тайминг (кто что берёт)

**Работаем ПАРАЛЛЕЛЬНО** — два трека почти не пересекаются по файлам:

| Трек | Владелец | Файлы | Пересечение |
|---|---|---|---|
| **UI-блоки 1–4, 6, 7** | разработчик | `templates/*.html`, `css/style.css`, `js/*.js`, точечно функции рендера в `generate.py` | низкое |
| **Генерация / контент / граф** | лид (я) | `gen_llm.py`, `generate.py` (пайплайн), `data/prompts/*`, `data/*.json`, картинки | низкое |

- **Разработчику можно брать сейчас:** блоки **1, 2, 3, 4, 6, 7** — они чистые UI, платного API не требуют,
  проверяются офлайн (`run.py html/links/check`). Начинать можно немедленно и параллельно.
- **Блок 5 (пиктограммы)** — сначала утвердить эскизы набора иконок с владельцем/дизайном; до этого
  разработчику НЕ начинать (иначе переделка). Помечен design-gated.
- **За лидом (я, не отдаём):** всё, что требует API/суждения/апробации — законы в промт генерации,
  картинки тегов/законов (FLUX-2-pro, серебряный стиль), gap-suggestions, презентационная пачка статей,
  консистентность графа знаний. Это НЕ UI и параллельно UI-блокам не мешает.

**Координация (чтобы не столкнуться):**
1. Финальный `python run.py html` (полная пересборка, asset_ver-бамп) — делает КТО-ТО ОДИН после того,
   как оба трека влили свои правки, а не каждый у себя (иначе конфликт на десятках тысяч файлов).
2. В `generate.py` не редактируем ОДНУ и ту же функцию одновременно — UI-трек трогает рендер-функции
   (`generate_section_page`, `parse_markers`-разметка в шаблоне и т.п.), контент-трек — пайплайн
   (`build_article`, `generate_*`, `backfill_*`). Списки функций — в разделах блоков.
3. Мелкие правки коммитим часто, полные реген-коммиты (много файлов) — согласованно.
