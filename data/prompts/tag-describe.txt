Создай образовательные мини-статьи для $n понятий физики, математики и астрономии.

Стиль: аналогии нужны, но БЕЗ шаблонных вводных фраз-клише («представьте себе», «вообразите» и
подобных) — сейчас так открывается чуть ли не каждое второе описание. Разнообразь подачу: прямое
сравнение без вводных слов, неожиданный факт, короткий вопрос, конкретный пример.

Лестница сложности СТРОГО: mini ⊂ simple ⊂ popular ⊂ advanced — каждый следующий уровень
добавляет детали, не противоречит предыдущему. На simple и popular уровнях, если используешь
физический термин без объяснения (например «квантовая суперпозиция», «спин», «инвариант») —
поясни его тут же в скобках парой слов, не отправляя читателя искать значение самому. На simple
это обязательно для КАЖДОГО непростого термина, на popular — где термин не был явно объяснён раньше.

Для каждого понятия создай:

MINI (мгновенная суть, 10 секунд, для человека без всякой подготовки):
- mini: 1-2 предложения, максимально образно, метафора/сравнение из повседневной жизни — без единого термина без объяснения
- practical_application: короткое «зачем это нужно / где встречается в жизни» — 1-2 предложения, конкретно (техника, природные явления, будущие технологии), не общими словами

SIMPLE (максимально просто, для школьника — это НАИБОЛЕЕ элементарный уровень из текстовых, после mini):
- description_simple: простые предложения с образными аналогиями из жизни/природы, каждый термин расшифрован в скобках
- history_simple: кто открыл и когда — коротко, простыми словами
- how_it_works_simple: как проявляется, с аналогией

POPULAR (средний, для любознательного взрослого неспециалиста — точнее и суше simple, но всё ещё без жаргона без пояснений):
- description_popular: чуть точнее и подробнее simple, без потери ясности
- fun_fact_popular: занятный факт простыми словами

ADVANCED (подробный, полный, для интересующегося физикой):
- description: развёрнутое описание
- history: история открытия — кто, когда, как развивалось (3-4 предложения). Упомяни связанных учёных если они есть в списке: $scientists.
- how_it_works: как это работает. Если понятие описывается одним из этих законов/принципов — упомяни его: $laws
- key_problems: ключевые нерешённые проблемы (массив строк)
- fun_fact: интересный факт (более специфичный/технический, чем fun_fact_popular)
- formulas: ключевые формулы (если есть) [{description, latex, meaning}]:
  - latex: валидный LaTeX с ОДИНАРНЫМИ слэшами (\frac{}{}, \hbar и т.д. — НЕ дублируй слэши, \\frac это ошибка), без окружения $$
  - meaning: расшифровка ВСЕХ переменных без исключений И всех констант с названием и численным значением (напр. «c — скорость света, c ≈ 3×10⁸ м/с»)

ОБЩИЕ поля (одни для всех уровней):
- id: английский ID (скопируй точно из входных данных)
- name: русское название СТРОЧНЫМИ буквами, кроме имён собственных и аббревиатур («чёрная дыра», «тёмная материя», но «телескоп Джеймс Уэбб», «JWST»)
- scientists: 0-3 имени учёных ТОЛЬКО из списка выше
- related_tags: 3-5 английских id ТОЛЬКО из: $all_ids

Понятия для обработки:
$tags_str

Ответь JSON-объектом:
{"tags": [{
