Создай развернутые образовательные карточки, полноценные описания для $n законов/принципов/теорем физики.

Стиль: аналогии нужны, но БЕЗ шаблонных вводных фраз-клише («представьте себе», «вообразите» и
подобных) — сейчас так открывается чуть ли не каждое второе описание. Разнообразь подачу: прямое
сравнение без вводных слов, неожиданный факт, короткий вопрос, конкретный пример.

Лестница сложности СТРОГО: mini ⊂ simple ⊂ popular ⊂ advanced — каждый следующий уровень
добавляет детали, не противоречит предыдущему. На simple и popular уровнях, если используешь
термин без объяснения (например «инвариант», «энтропия», «квантовое состояние») — поясни его
тут же в скобках парой слов. На simple это обязательно для КАЖДОГО непростого термина.

Формулы ОДНИ И ТЕ ЖЕ для всех уровней, различается только текст описания. Для каждого закона:

MINI (мгновенная суть, 10 секунд, для человека без всякой подготовки):
- mini: 1-2 предложения, максимально образно, метафора/сравнение из повседневной жизни — без единого термина без объяснения
- practical_application: короткое «зачем это нужно / где встречается в жизни» — 1-2 предложения, конкретно (техника, природные явления, инженерия), не общими словами

SIMPLE (максимально просто, для человека без спец подготовки — это НАИБОЛЕЕ элементарный текстовый уровень после mini):
- description_simple: простые предложения с аналогиями, чтобы было понятно даже неподготовленному, каждый термин расшифрован в скобках
- how_it_works_simple: где проявляется, примеры из жизни

POPULAR (средний, для любителя науки — точнее и суше simple, но всё ещё без необъяснённого жаргона):
- description_popular: история открытия и суть закона с аналогией, точнее и подробнее simple
- fun_fact_popular: занятный факт простыми словами

ADVANCED (полный, образовательный, для интересующегося физикой):
- description: история открытия, строгая формулировка и смысл
- history: ИСТОРИЯ ОТКРЫТИЯ — кто, когда, как пришли к закону, кто развивал (3-5 предложений). Обязательно называй учёных; если они есть в списке — пиши точно как там: $scientists.
- how_it_works: как применяется, границы применимости (3-4 предложения)
- key_problems: нюансы/ограничения/открытые вопросы (массив строк)
- fun_fact: интересный факт (более специфичный/технический, чем fun_fact_popular)

ОБЩИЕ поля:
- id: английский ID (скопируй точно из входных данных)
- name: русское название СТРОЧНЫМИ буквами, кроме имён собственных («закон всемирного тяготения», «второй закон термодинамики», но «теорема Нётер», «эффект Доплера»)
- formulas: 1-3 ключевые формулы этого закона [{description, latex, meaning}]:
  - latex: валидный LaTeX с ОДИНАРНЫМИ слэшами (\frac{}{}, \hbar, \partial и т.д. — НЕ дублируй слэши, \\frac это ошибка), без окружения $$
  - meaning: расшифровка ВСЕХ переменных из формулы без исключений (что означает каждый символ) И всех констант с их названием и численным значением (напр. «ħ — редуцированная постоянная Планка, ħ ≈ 1.0546×10⁻³⁴ Дж·с»)
- scientists: имена учёных-первооткрывателей (те, кого назвал в history; из списка выше если есть)
- tags: скопируй список tags из входных данных
- related_laws: 0-3 id других законов из этого списка: $all_ids

Законы для обработки:
$laws_str

Ответь JSON-объектом:
{"laws": [{

  "description_popular": "...", "fun_fact_popular": "...",

  "formulas": [{"description": "Сила тяготения", "latex": "F = G\\frac{m_1 m_2}{r^2}", "meaning": "..."}],

}]}
