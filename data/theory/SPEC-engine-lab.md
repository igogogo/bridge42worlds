## Что показывает
«Вся термодинамика одним взглядом»: 4 базовых процесса идеального газа + рабочий цикл двигателя на P–V.

### Параметры (`params` в JSON)
1. `mode` — выбор режима. Реализовать как набор кнопок-табов НАД стендом (не слайдер): 
   `isothermal | adiabatic | isobaric | isochoric | engine`. (Слайдер `explorable` не подходит для дискретного выбора —
   сделать свои кнопки в странице, при клике пересобирать/обновлять cfg или держать `mode` во внешней переменной,
   которую читают animate/derive/formula/plot; см. как в lab-* хранится `LANG`.)
2. `drive` — ведущий параметр процесса, слайдер: для isothermal/adiabatic/isobaric ведём **объём V** (1..10 л),
   для isochoric ведём **давление/температуру**. Проще: один слайдер `V` (1..10 л), а для изохоры вести `T`.
   Допустимо сделать 2 слайдера: `V` (1..10 л) и `T` (200..1200 K) и в каждом режиме «активен» тот, что физичен
   (в изотерме двигаем V при фикс. T; в изохоре двигаем T при фикс. V; и т.д.) — заблокированный можно не блокировать,
   просто он не влияет на связанную величину по закону режима.

### Константы (JSON `constants`)
`R=8.314`, `n=0.4` (моль), `gamma=1.4` (двухатомный газ), опорные `V0=5` л, `T0=300` K, `P0 = n*R*T0/V0`.

## Физика (код в странице)
Базис: `P = nRT/V` (кПа при V в л, R=8.314). Для каждого режима — свой инвариант:
- **isothermal** (T=const): `P·V = const`. Кривая на P–V — гипербола. Работа `W = nRT·ln(V2/V1)`.
- **adiabatic** (без теплообмена, Q=0): `P·V^γ = const`, `T·V^(γ-1) = const`. При сжатии газ САМ греется
  (частицы краснеют — важный визуальный инсайт, что нагрев без плитки). Кривая круче изотермы.
- **isobaric** (P=const): двигаем V → `T = P·V/(nR)`. Прямая горизонталь на P–V. `W = P·ΔV`.
- **isochoric** (V=const): двигаем T → `P = nRT/V`. Вертикаль на P–V. `W = 0` (объём не меняется).
- **engine** — замкнутый цикл. Взять **цикл Отто** (бензиновый ДВС): 2 адиабаты + 2 изохоры, либо **Карно**
  (2 изотермы + 2 адиабаты). Рекомендую Отто как более «двигательный». Точка бежит по петле по кругу (анимация по времени).
  - Работа за цикл `W = площадь петли` (интеграл ∮P dV) — можно считать численно по вершинам.
  - КПД: Отто `η = 1 − 1/r^(γ−1)`, где `r = Vmax/Vmin` (степень сжатия); Карно `η = 1 − Tc/Th`.
  - Показать η числом и залить площадь петли (это и есть полезная работа).

## Стенд (`stage.draw`)
Переиспользовать визуал газа: **сосуд с поршнем + частицы** (как в `lab-gas.html`), поршень двигается по текущему V,
частицы краснеют по T (в адиабате при сжатии — сами). В режиме `engine` поршень ходит туда-сюда по тактам цикла.
Высота сцены ~250.

## График (`plot`) — P–V диаграмма (главное!)
- x = V (1..10 л, `xV`), y = P (кПа, `yP`), диапазоны функциями от state (как в lab-gas).
- `curve`: рисует линию ТЕКУЩЕГО процесса через опорную точку (для engine — весь замкнутый цикл; можно рисовать
  цикл отдельной отрисовкой поверх, а `curve` вернуть текущую ветку).
- `marker`: текущая точка (V,P). В engine — бегущая по петле точка + заливка площади (работа).

## Формула (`formula`) — с подстановкой чисел
Показывать активный закон режима с числами и результат, напр.:
- изотерма: `P·V = 0.4·8.314·300 = 997 → P = 997/V`
- адиабата: `P·V^1.4 = const · T·V^0.4 = const` + текущие T,P
- engine: `η = 1 − 1/r^0.4 = 1 − 1/6^0.4 = 51%` + `W = <площадь> Дж за цикл`
Плюс мульти-единицы как в газе (P: кПа/атм/бар; T: K/°C/°F) — можно живой строкой под стендом.

## Приёмка (как проверять)
- Переключение режимов меняет кривую на P–V и активный закон в формуле.
- Адиабата круче изотермы; при сжатии в адиабате T (и цвет частиц) растёт без «плитки».
- engine: точка идёт по замкнутой петле, площадь = работа, η считается и меняется от степени сжатия.
- Локаль ru/en переключается, подписи и единицы переводятся.
- Нет краша при «время назад»/сбросе (dt зажат `Math.max(0, Math.min(0.05, t-last))`).
