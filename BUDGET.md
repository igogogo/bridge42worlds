# BUDGET.md — журнал баланса (DeepSeek/DeepInfra)

Живой учёт трат для бюджетирования продакшена. Ведём **CNY-баланс** (это кошелёк, который
реально тратится текущими прогонами — DeepSeek chat/translate calls списывают именно с него,
USD ни разу не двигался при живом API-расходе, даже когда CNY заметно падал). USD-баланс
($22.37 на 2026-07-13, после доп. пополнения +$20) юзер подтвердил как ОБЩИЙ бюджет — это тоже
наши деньги, доступные при необходимости (напр. как резерв/на конвертацию), просто API их пока
не трогает автоматически. В журнал пишем только движение CNY, но при оценке «сколько ещё есть
на всё» держим в уме оба числа.

Как обновлять: после каждого платного прогона (регенерация статей, рост
tags/laws/scientists, картинки FLUX и т.д.) — просим у юзера свежий баланс, считаем
дельту с прошлой записи, пишем что именно её вызвало (команда + масштаб).

## Заметки

- Первые 4 строки восстановлены по ходу разговора (скриншоты баланса без чёткой привязки
  к конкретному коммиту/команде на момент записи) — держим их как ориентир, не как точный аудит.
- ⚠️ Инцидент в рамках батча 232.84: `run.py tags --force --refine` (без `--gaps`) неожиданно
  вызвал слепую ПЕРЕЗАПИСЬ активного списка тегов (`tag_list.py` без `--gaps` не топ-ап, а полная
  перегенерация с нуля — баг, не относящийся к `--force`, который должен был трогать только
  описания). Все 11 тегов из пробел-анализа (п.3 выше) были стёрты и заменены новым слепым
  списком той же длины (60) — их стоимость генерации/перевода фактически потрачена впустую.
  Причина найдена и исправлена в `run.py::cmd_tags` (список больше не трогается без `--gaps`/
  `--rebuild`, если уже существует) — не повторится. Законы/учёные этим багом не затронуты
  (их list-скрипты изначально были инкрементальным топ-апом, не слепой перезаписью).
- Следующий шаг роста справочников (если нужен) — ещё раунд(ы) `run.py evolve --rounds N`,
  до потолков `growth.*_max` в config.json (150/150/100/100) — сейчас на 70/72/60/60, есть запас.
- Перегенерация обрезанных аннотаций (fix для обрезки — см. `gen_llm.py` `_cap_text`) выполнена
  2026-07-12 — но ТОЧЕЧНО (57 реально обрезанных статей батча 07-01…07-06 через
  `abstract_backfill_targeted.py`), не блинд `run.py abstracts --force` по всем 240 (которое зря
  пережевало бы ещё ~180 уже нормальных статей).
- Экспресс-режим подтверждён на практике заметно дешевле полного цикла — хороший инструмент
  для наращивания объёма архива, если нужно больше контента при том же бюджете (см. строку 199.12).
- ⚠️ Дважды словили зависший LLM-запрос (0% CPU, 10+ минут без ответа/ошибки) при разных прогонах
  2026-07-12 — SDK-клиент был без явного `timeout`, дефолт httpx может ждать до 10 минут. Добавлен
  `timeout=180` в `common.py::client` — впредь зависший запрос сам упадёт и уйдёт в обычный ретрай
  `chat()` (10с→30с→30с бэкофф), а не будет висеть неопределённо.
